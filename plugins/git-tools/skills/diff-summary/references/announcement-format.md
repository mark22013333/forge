# 版本公告 JSON 格式規範

## 格式

```json
{
  "version": "vX.X.X",
  "date": "YYYY-MM-DD",
  "items": [
    "以使用者角度描述的功能變更說明"
  ]
}
```

---

## 欄位說明

| 欄位 | 說明 |
|------|------|
| `version` | 版本號，留佔位符讓使用者自行填寫 |
| `date` | 日期，留佔位符讓使用者自行填寫 |
| `items` | 陣列，每項用非技術人員能理解的語言描述變更對使用者的影響 |

---

## 撰寫原則

- 從**使用者角度**描述，不提及程式碼細節（類別名、方法名）
- 使用**繁體中文**
- 每項以動詞開頭（新增、優化、修復、調整）
- 強調對使用者的影響與好處
- 純重構或內部修復，寫「系統內部優化，提升穩定性與效能」

---

## 範例

### 功能新增型

```json
{
  "version": "vX.X.X",
  "date": "YYYY-MM-DD",
  "items": [
    "匯入功能大幅升級：支援上傳包含圖片的 ZIP 壓縮檔，可一次性批量建立物品資料與圖片，無需逐筆上傳。",
    "新增「匯入範本懶人包」下載，內含 Excel 範本、範例圖片與詳細操作說明文件，資料整理更上手。",
    "優化匯出功能，現在可匯出包含物品圖片的完整資料備份，並提供即時的進度顯示。",
    "匯入結果報告更清晰，系統將詳細列出成功、失敗筆數以及圖片處理狀況（如圖片遺失或覆蓋提示）。",
    "新增系統自動維護機制，定期清理過期的備份檔案，確保系統運作效能。"
  ]
}
```

### Bug 修復型

```json
{
  "version": "vX.X.X",
  "date": "YYYY-MM-DD",
  "items": [
    "修復推播訊息在特定時段發送失敗的問題，現在所有排程推播皆能準時送達。",
    "修正統計報表中開封數與點擊率數據不一致的問題，數據更加準確可靠。"
  ]
}
```

### 純重構型

```json
{
  "version": "vX.X.X",
  "date": "YYYY-MM-DD",
  "items": [
    "系統內部優化，提升穩定性與效能。"
  ]
}
```

### 混合型

```json
{
  "version": "vX.X.X",
  "date": "YYYY-MM-DD",
  "items": [
    "新增里公布欄統計數據加註排程更新說明文字，方便里長了解資料更新時間。",
    "修正訂閱推播開封數欄位與點擊率公式不一致的問題。",
    "優化數據分析統計邏輯，調整排程執行時間以減少尖峰時段負載。"
  ]
}
```
